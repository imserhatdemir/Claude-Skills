Review the landing page in $ARGUMENTS (or current file/selection if not specified).

**Step 1** — Read the landing page code and identify the CTA, hero section, and content structure.

**Above the Fold**
- Headline clearly communicates the value proposition in one sentence
- Subheadline supports and expands the headline
- CTA is visible without scrolling, action-oriented, and singular
- Hero image/video reinforces the message, not decorative noise

**Conversion Optimization**
- Single primary CTA per section; no competing actions
- CTA copy is specific ("Start Free Trial" > "Get Started" > "Submit")
- Social proof (testimonials, logos, user counts) placed near CTAs
- Friction points in signup/purchase flow identified

**Content Structure**
- Problem -> Solution -> Proof -> CTA narrative flow
- Features presented as benefits (user outcome, not technical detail)
- FAQ section addresses real objections
- Pricing section (if present) is clear, no hidden fees implied

**Visual Hierarchy**
- Clear H1 > H2 > body type scale
- Whitespace used to separate sections, not fill space
- Consistent color usage: primary for CTAs, neutral for content

**Trust Signals**
- Security badges, guarantees, or certifications visible near purchase
- Real testimonials with name, photo, and role/company

**Performance & SEO**
- Meta title and description present and compelling
- OG tags for social sharing
- Images have alt text
- LCP element identified and optimized

**Mobile Landing Experience**
- CTA reachable with thumb on mobile
- No horizontal scroll
- Font sizes readable without zooming (min 16px body)
- Hero section not overcrowded on small screens

Output findings as a prioritized list: **Critical > Major > Minor**. Include `file_path:line_number`. Suggest a concrete fix for Critical and Major items.
